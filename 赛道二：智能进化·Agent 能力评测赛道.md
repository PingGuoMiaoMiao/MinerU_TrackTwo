# 赛道二：智能进化·Agent 能力评测赛道

# 赛题介绍

## 赛题背景

随着大模型与数据处理技术的发展，Data Agent 正逐渐成为数据智能系统的核心形态，其能力不再局限于单一模型推理，而是需要在复杂任务中完成**规划、工具调用、多步骤执行与结果验证**等行为。本赛道围绕真实产业中海量数据处理与语料加工的典型需求，鼓励参赛团队基于MinerU开发和提交能解决实际场景问题、高效且高质量处理能力以及低成本的数据智能体Data Agent，通过赛事筛选和选拔在语料生产加工领域表现最出色的Agent技术方案，为行业树立技术标杆。

## 赛题任务

参赛团队需要开发和提交一个具备自动化数据处理能力的 Data Agent数据智能体，能够在复杂任务环境中自动理解任务需求、调用工具或模块完成数据处理、生成结构化结果以及输出可验证的处理日志与结果。系统需支持通过 API 接口提交至组委会进行能力评测。
Data Agent 数据处理的类型主要面向文档或网页（如PDF、WORD、PPT、HTML等），能够系统性的解决当下单一模型或数据处理管线面临的数据处理难点，难题包括但不限于：财务报表中密集数字解析的准确性和幻觉问题、跨页合并与全局指代消解、工程图或流程图解析、低质量文档的鲁棒性（如严重模糊、光线不均的拍照件、手写签章重叠等极端场景）、复杂图表解析、基于规范或行业标准的解析等，由参赛团队根据当前技术的难点自行定义解决的问题。

## 赛题说明

参赛团队设计和研发Data Agent以解决某类或多类当下数据处理的难点，在满足成本、速度与精度之间最佳平衡的同时需要满足以下三种能力：

- 任务一：数据理解与结构化处理 ，即从多源数据中识别数据结构，完成信息抽取、格式转换、数据清洗及元数据抽取等操作，并输出统一规范的结构化数据结果。
- 任务二：复杂任务规划与自动执行，即面对复杂的任务目标后，能自动将其“拆解”成多步子任务，能合理调用工具完成数据处理流程，典型场景包括数据转换、批处理执行、任务调度以及异常处理与恢复等。
- 任务三：系统稳定性与综合能力评测， 系统需要在复杂或高负载条件下保持稳定执行，并确保结果输出的可靠性与一致性。该任务旨在模拟实际生产环境中的应用场景，全面检验系统的工程成熟度与可落地能力。

## 提交内容

**参赛队伍需提交以下内容：**

1. 鼓励参赛队伍开放系统实现代码或关键模块代码，提交Data Agent系统代码或关键模块代码（开源至 GitHub 并提交 repo 链接）
2. 系统部署与运行说明文档：提供完整的Data Agent部署与运行指南，包括系统架构、运行环境要求（操作系统、依赖库、硬件资源等）、启动方式、测试方法及日志查看方式，确保系统能够被复现与验证。或提供互联网可访问的服务接口信息，包括系统服务地址或 API 接口说明文档，明确接口调用方式、参数说明、返回结果格式及认证方式，以支持组委会进行验证。
3. 完整的技术报告文档，包括 Data Agent 的整体设计方案、任务执行机制、数据处理与工具调用能力、系统性能与稳定性说明，并提供不少于 5 个典型任务执行示例，以及系统适用场景与应用价值说明。
4. 系统运行日志与测试结果：提供系统测试任务相关文件，且提供系统在测试任务执行过程中的运行日志或结果记录，用于验证系统执行过程的完整性与可追溯性；日志内容应包含任务输入、执行步骤、调用工具信息及最终输出结果等关键字段。
5. 其他相关材料（加分项：PPT、技术文档、演示视频等形式对系统能力或应用成果的介绍）

**具体评审分为五大维度：**

1. 复杂文档理解与结构化处理能力（20 分）
   重点评估 Agent 对文档的解析与结构化能力。考察系统能否精准应对当前文档解析的难点痛点。关注信息抽取的准确性、格式转换的规范性以及数据清洗的质量，确保输出高标准、稳定的结构化结果。
2. 难点场景攻克与技术创新性（15 分）
   重点评估方案在解决实际数据处理痛点中的技术创新与应用价值。考察团队自定义的难题是否切中行业痛点，以及所采用的 Agent 架构、数据处理管线或算法设计是否具备显著的原创性与先进性，能否为语料加工领域树立标杆。
3. Agent 任务规划与自动执行能力（30 分）
   核心评估 Agent 的自主决策与流转能力。考察系统能否精准理解复杂指令，自动将目标拆解为多步子任务，并合理调用工具（如MinerU等）完成流程。重点关注多步骤执行的逻辑性、异常报错的自动恢复能力，以及系统在成本、处理速度与精度之间的最佳平衡。
4. 系统稳定性与工程可复现性（20 分）
   重点评估系统在复杂高负载环境下的工程成熟度。考察系统运行是否稳定、输出是否可靠一致。关注提交的部署文档、API接口及运行日志是否完整详实，执行步骤与工具调用记录是否高度可追溯，确保系统具备极强的可复现性与实际生产环境的可落地能力。
5. 代码开源共享与产业生态价值（15 分）
   重点评估项目的开源贡献度与产业推广潜力。考察团队是否在 GitHub 等平台规范开源核心代码，技术报告与测试用例是否详尽。关注该 Data Agent 在海量语料生产、行业规范解析等真实产业场景中的泛化能力、应用广度，以及对大模型数据生态的积极影响。
