# 演示讲稿与 PPT 大纲

## PPT 大纲

### 1. 项目定位

标题：复杂文档处理 Data Agent

要点：

- 面向 PDF、Word、PPT、HTML、TXT 和网页 URL 的统一结构化处理。
- 自动完成任务理解、规划、工具调用、抽取、校验和日志归档。
- 重点解决复杂文档解析、跨格式统一输出和过程可追溯问题。

### 2. 赛题痛点

要点：

- 财报、合同、研究报告等文档格式复杂，人工处理成本高。
- 单一模型容易出现跨页遗漏、表格抽取不稳、证据不可追溯。
- 生产环境需要稳定、可复现、可审计的自动化流程。

### 3. 系统架构

要点：

- API 层：FastAPI 接收文件、URL 或文本任务。
- Agent 层：规划器拆解任务，执行器调度解析和抽取。
- 工具层：MinerU SaaS、MinerU Agent、本地 MinerU CLI 和本地兜底解析。
- 存储层：JSON 保存任务状态，JSONL 保存可追溯事件日志。

### 4. MinerU 工具链接入

要点：

- `mineru_precise`：SaaS 精准解析，适合正式评测。
- `mineru_agent`：精准接口失败时自动降级。
- `mineru_local`：开源本地 CLI，可展示可复现工具链。
- 统一读取 Markdown 中间表示，交给 Agent 做结构化抽取。

### 5. 现场演示流程

命令：

```bash
python -m uvicorn src.main:app --host 127.0.0.1 --port 8000
python scripts/run_smoke_tests.py --no-start
```

接口演示：

```bash
curl http://127.0.0.1:8000/health
curl http://127.0.0.1:8000/v1/tasks/{task_id}
curl http://127.0.0.1:8000/v1/tasks/{task_id}/logs
```

讲解重点：

- 创建任务后立即返回 `task_id`。
- 后台异步执行解析、抽取和校验。
- 结果中包含 `summary`、`metadata`、`entities`、`tables`、`quality`、`evidence`。
- 日志中保留任务输入、计划步骤、工具调用和最终质量检查。

### 6. 测试结果

要点：

- 本地 5 类样例：TXT、HTML、DOCX、PPTX、PDF 全部成功。
- 公开网络 PDF：arXiv、NASA 报告、NASA 技术论文、DigitalOcean 年报全部成功。
- 132 页年报解析字符数超过 50 万，证明系统可处理长文档。
- 本地 MinerU CLI 已完成 arXiv PDF 解析验证。

### 7. 工程稳定性

要点：

- 上传大小限制，防止异常输入拖垮服务。
- 任务状态与事件日志分离，便于查询和审计。
- LLM 不可用时返回 fallback 结构，保证接口稳定。
- MinerU 精准解析失败后自动降级，必要时可切换本地 CLI。

### 8. 应用价值

要点：

- 可用于语料生产、财报抽取、合同审查、行业报告解析和网页公告结构化。
- 输出 schema 稳定，方便进入后续评测、知识库或数据生产流水线。
- 工具链可替换，可接入团队自有模型与更多 OCR/表格识别能力。

## 3 分钟讲稿

大家好，我们的项目是一个面向复杂文档处理的 Data Agent。它支持文件、网页 URL 和直接文本输入，能够自动理解任务目标，规划执行步骤，调用 MinerU 和本地解析工具，并输出结构化 JSON 和完整执行日志。

这个赛题的核心难点是，真实文档往往包含跨页文本、复杂表格、合同条款和长篇报告。单一模型直接读取容易遗漏信息，也很难证明结果来自哪里。因此我们把系统拆成五个阶段：输入解析、任务规划、结构化抽取、质量校验和日志归档。

在工具链上，我们接入了三种 MinerU 后端。正式评测推荐使用 `mineru_precise`，它会调用 MinerU SaaS 精准解析；如果精准接口异常，系统会降级到 `mineru_agent`；如果需要展示开源可复现链路，可以切换到 `mineru_local`，通过本地 MinerU CLI 生成 Markdown、content list 和 middle/model JSON。

现场演示时，我们先启动服务，再提交一条任务。任务创建接口会立即返回 `task_id`，后台异步完成解析和抽取。查询任务结果可以看到摘要、元数据、实体、表格、质量检查和证据片段。查询日志接口可以看到排队、规划、工具调用和任务成功事件，整个执行过程是可追溯的。

目前我们已经完成了两组测试。本地样例覆盖 TXT、HTML、DOCX、PPTX 和 PDF，全部成功；公开 PDF 覆盖 arXiv 论文、NASA 报告和 132 页 DigitalOcean 年报，也全部成功。长文档年报解析字符数超过 50 万，说明系统可以处理较大规模的真实材料。

总结来说，这个 Data Agent 的特点是统一入口、可替换工具链、稳定 schema 输出和可审计日志。它可以直接用于财报抽取、合同解析、行业报告结构化和语料生产流水线，也方便继续扩展 OCR、复杂表格识别和自动评测指标。
